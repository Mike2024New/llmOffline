# 👨‍🎓 О проекте

**llmOffline** - автономный клиент для общения с заранее скачанными .gguf моделями. Если понадобится скачать модель
то потребуется явное согласие, без которого компонент не полезет в web.

> Кратко - легкий переносимый компонент который может быть как самодостаточным так и интегрированным в системы, являсь
> лишь одним звеном цепи. Задача компонента обрабатывать входящие запросы и выполнять функции предусмотренные llm.
> Компонент постоянно дорабатывается в ближайшем будущем ожидается webRTC и fastapi интерфейсы, для интеграции с другими
> системами.

## 🚀 Что умеет

- 🔗 **Скачивать .gguf модели** - с hugging-face, на основании схемы репозитория указаваемой в `models_map.json`, схема
  может расширяться по любые модели которые есть на HF. В случае прерывания программы закачка будет продолжена с места
  остановки. Для загрузки моделей требуется подтверждение на разрешение выхода в интернет.

- 🤖 **Вести интерактивный чат в режиме вопрос ответ** - чат ведется без хранения истории. Контекст модели ограничен
  текущим запросом, такой режим может подойти например для переводчика на другой язык.

- 🤖 **Вести интерактивный чат в режиме диалога** - хранит историю сообщений (её размер задается в настройках в
  `settings.json`).

- 💻 **Компонент легко разворачивается на любой машине.** - созд. проект с виртуальным окружением, клонировать туда этот
  репозиторий запустить `python start.py` и проект разворачивается.

- 📦 **Сборка в .exe(bin)** - через cli.py можно собрать бинарный файл `python cli.py build`

- 📖 **Вести структурированный (машиночитаемый) лог** о выполненных операциях корневая папка logs команда `folder`, чтобы
  открыть корневую папку приложения.

---

## ⚠️ Важно

- Для работы оффлайн модели необходимо загрузить заранее.
- Не делать первичную инициализацию через `uv sync`, для развертывания использвать скрипт `python start.py`, он скачает
  подходящий whl для llama.cpp.

## ⚡ Быстрый старт

### На windows:

```bash
git clone https://github.com/Mike2024New/llmOffline.git
cd llmOffline # Перейти в папку с архивом
python -m venv .venv # Создать venv (! Папка должна называться .venv, минимальная версия 3.12)
.venv/Scripts/activate # Активировать виртуальное окружение:
python start.py # Запустить стартовый скрипт который скачает зависимости проекта.
python cli.py --help # Войти в интерфейс приложения.
# 7. Опционально: для получения исполняемого файла (bin/exe):
python cli.py build # сборка .exe
# * - откроется папка с собранным дистрибутивом, который можно прописать в системных переменных
```

#### Прописать в переменных средах(если нужен доступ по имени):

```text
После команды `python cli.py build`, откроется папка с проектом, и там будет файл `llm.exe`, прописать путь этой
к этой папке в переменных средах, например C:\Users\<ваше_имя>\llmOffline\releases\win\dist ** Перезагрузить терминал **.
```

#### Использование (допустим бинарный файл назвали llm и прописали в системных переменных):

```bash
# запуск интерактивного чата с доступными моделями
# запросит подтверждение на разрешение скачивания моделей с HF если их нет. 
llm chat
# выход из чата после нажатия enter
# подробную справку см. в 
llm --help
```
